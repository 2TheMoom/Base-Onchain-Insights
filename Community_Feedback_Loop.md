# 💬 Community Feedback Loop: Social Engagement Analyzer

Building in the open means listening to the "Based" community. This document outlines how we collect and implement user feedback.

### 1. Bug Reporting (The "Debug" Channel)
* **GitHub Issues:** The primary place for technical bugs (e.g., "Rabby Wallet connection failed on Brave").
* **Priority:** Issues affecting "Wallet Connectivity" or "Score Calculation" will be addressed within 48 hours during Beta.

### 2. Feature Requests (The "Based" Wishlist)
* **Warpcast/Farcaster:** Users can tag the project or use the #BasedScore hashtag to suggest new metrics (e.g., "Add NFT transaction volume").
* **Voting:** High-engagement suggestions will be added to the Phase 4 roadmap.

### 3. "Based" Office Hours
* **Open Contributions:** We encourage other developers to fork the repo and submit Pull Requests for UI improvements or SQL optimizations.
* **Recognition:** Contributors will receive a special "Contributor" badge on their Based Profile Card.

### 4. Implementation Cycle
* **Review:** Feedback is reviewed every weekend.
* **Update:** Valid bug fixes are committed directly to the main branch with clear documentation on what was changed.
